# 基本設計書作成コマンド (/software-design)

プロのWeb開発ディレクターとして、要件定義書を元に基本設計書を作成してください。

## 前提条件
- 要求定義書（./document/001_要求定義.md）を参照する
- 要件定義書（./document/002_要件定義.md）を参照する
- エンジニアが詳細設計・実装に進めるレベルの設計書を作成する
- クリックゲームWebサイトの特性に応じた設計を行う

## 作成指針
1. 既存の要求定義書・要件定義書を読み込み、内容を理解する
2. Webアプリケーションの基本設計として必要な要素を整理する
3. 実装に必要な技術選定と設計方針を明確化する
4. エンジニアが迷わず実装できる詳細度で記載する
5. **設計書にはコード例を含めない** - 仕様と設計方針の記述に徹する

## 設計書構成
基本設計書は以下の構成で作成してください：

1. **システム構成設計**
   - 作業指示者へのヒアリング実施
     ※以下の項目について一問一答形式で確認してください。
     　要求定義書・要件定義書から読み取れる情報を基に、プロジェクトに適した選択肢や推奨案を提示した上で確認を行う：
     - 希望する技術スタック（フロントエンド・バックエンド）
     - 開発環境・開発ツール
     - インフラ・デプロイ環境
     - 外部サービス・ライブラリ使用の可否
     - その他技術的制約や要望
   - アーキテクチャ概要
   - 技術スタック選定
   - ディレクトリ構成

2. **データ設計**
   - データモデル定義
   - データベース設計（必要に応じて）
   - データ永続化方針
   - 状態管理設計

3. **画面設計**
   - 画面一覧と役割
   - 画面レイアウト設計
   - ワイヤーフレーム（各画面・デバイス別）
   - UI/UXコンポーネント設計（設計方針・概要レベル）
   - 画面遷移図
   - フロントエンド処理フロー
   - クライアントサイドイベント設計

4. **バックエンド処理設計**
   - サーバーサイド業務フロー
   - API設計（必要に応じて）
   - データ処理ロジック
   - バッチ処理設計（必要に応じて）

5. **外部連携・リソース設計**
   - 外部API連携（必要に応じて）
   - 静的リソース管理
   - セキュリティ設計
   - 非機能要件への対応

※プロジェクトの特性に応じて、不要な項目は省略し、必要な項目は追加・詳細化してください。

## 記載上の注意事項
- **設計書として適切な記述**: 具体的なコード実装例は記載しない
- **仕様中心の記述**: データ型、処理方針、設計パターンを文章で説明
- **技術選定の根拠**: なぜその技術を選択するかの理由を明記
- **実装の指針**: どのような方針で実装すべきかを記述

## 出力形式
基本設計書は ./document/003_基本設計.md として保存してください。
